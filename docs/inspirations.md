# Inspirations

This document records principles rather than assets to copy. Specific visual
references can be added when they are supplied.

## Engineering notebooks

Useful qualities: visible structure, concise annotations, and evidence of a thought
being developed. The profile should borrow their clarity without imitating roughness
for effect.

## Technical documentation

Useful qualities: predictable hierarchy, direct language, navigable detail, and an
assumption that the reader's time matters. Supporting documents should reward deeper
reading while the README remains brief.

## System diagrams and blueprints

Useful qualities: relationships made visible through alignment, connection, and
scale. Diagrams should represent an idea rather than serve as technical-looking
wallpaper.

## Terminals

Useful qualities: economy, focus, and high information density. Literal terminal
prompts, faux command output, and neon “hacker” styling are outside the visual
direction.

## Swiss-style typography

Useful qualities: strong grids, deliberate whitespace, restrained type, and clear
hierarchy. GitHub Markdown limits typographic control, so composition and editing
should carry most of this influence.

## Lo-fi espresso

Useful qualities: warm dark neutrals, soft contrast, a little grain, and the calm of a
late-night notebook beside a cup of coffee. The profile borrows the mood rather than
adding literal coffee illustrations. Crema, caramel, and muted sage carry the ambient
color; brand marks keep their official colors.

## Portal and Half-Life

Portal is present first as a personal favourite, not as a technical credential. Its
puzzles also fit the profile's wider interest in curiosity and discovery. The playful
replay loop in `practice-strip-espresso.gif` uses original paired-gateway geometry without
reproducing logos, interfaces, characters, or other game assets.

## GitHub profile ecosystem

The [`github-profile` topic](https://github.com/topics/github-profile) was reviewed for
GitHub-native presentation patterns. The useful ideas are structural: one clear hero,
small social navigation, restrained technology marks, and strong mobile hierarchy.
The public README keeps each category to one compact logo row.

The profile intentionally does not adopt the topic's common generated features. Stats,
streaks, trophies, counters, badge collections, activity feeds, animated headers, and
template-style “about me” blocks add noise without helping the introduction.

The [profile header generator](https://github.com/leviarista/github-profile-header-generator)
reinforced the value of a clear title, subtitle, careful alignment, and light/dark
review. The resulting banner remains original and repository-owned rather than being
generated by an external service.
